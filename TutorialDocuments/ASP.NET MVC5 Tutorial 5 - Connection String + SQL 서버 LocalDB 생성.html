Connection String + SQL 서버 LocalDB 생성

ASP.NET MVC5 Tutorial 4 - Model 추가 에서 생성했던 UserInfoDBContext 클래스는 DB의 연결, UserInfo Object <-> DB Mapping 작업을 가능하게 합니다.  y
한가지 의문점을 가져 볼 만한건, 우리는 아무런 DB셋팅도 하지 않았는데 어떻게 DB에 연결 되는걸까요? 

바로, Entity Framework 가 LocalDB 를 사용하기 때문입니다.

이번 Tutorial에서는 Web.config 파일을 이요해서 연결 문자열(Connection String)을 추가 해보도록 하겠습니다.

SQL Server Express LocalDB

LocalDB는 사용자측에서 돌아가는 가벼운 SQL Server Express DB 라고 보시면 됩니다. LocalDB는 .mdf 파일을 사용해, SQL Server Express의 특수 모드에서 일반적인 DB와 같이 동작합니다. 
일반적으로 LocalDB는 WebProject의  App_Data 폴더에 생성 됩니다.

SQL Server Express는 단순히 개발 혹은 공부 목적으로만 사용 하시길 권장 합니다. IIS에서 작동하도록 설계되어 있지도 않기 때문에 일반 웹 Application 용도로 사용하시면 안됩니다.
그래도 LocalDB는 SQL Server나 SQL Azure로 쉽게 마이그레이션 가능합니다.

Visual Studio 2013( 2012 ) 에서는 LocalDB가 설치가 Default 값입니다.
기본적으로, Entity Framewok는 Class 이름과 같은(현재 프로젝트에서는 UserInfoDBContext) DB 연결 문자열(Connection String)을 갖습니다. 자세한 내용은 SQL Server Connection Strings for ASP.NET Web Applications 을 참조하세요.



Solution Explorer의 Web.Config 파일을 열어보세요. ( View 폴더 트리에 있는 Web.Config 파일이 아니니 주의 하세요! )



그리고 <connectionStrings>  항목을 찾아 다음과 같은 항목을 추가 해 보세요.

<add name="UserInfoDBContext" 
   connectionString="Data Source=(LocalDb)\MSSQLLocalDB;AttachDbFilename=|DataDirectory|\UserInfo.mdf;Integrated Security=True;Integrated Security=True" 
   providerName="System.Data.SqlClient" 
/> 
수정된  Web.config 파일의 <connectionStrings> 입니다.
  <connectionStrings>
    <add name="DefaultConnection"
         connectionString="Data Source=(LocalDb)\MSSQLLocalDB;AttachDbFilename=|DataDirectory|\aspnet-GameServer-20151207030844.mdf;Initial Catalog=aspnet-GameServer-20151207030844;Integrated Security=True"
         providerName="System.Data.SqlClient" />
    <add name="UserInfoDBContext"
         connectionString="Data Source=(LocalDb)\MSSQLLocalDB;AttachDbFilename=|DataDirectory|\UserInfo.mdf;Integrated Security=True;Integrated Security=True"
         providerName="System.Data.SqlClient" />
  </connectionStrings>
두 연결 문자열( Connection Strings )을 살펴보면 꽤 비슷합니다.

하나는 DefaultConnection 이라는 이름으로, Application에 접근 가능한 사용자를 제어하기 위한 멤버쉽 DB에 사용 됩니다.
우리가 방금 추가 한 문자열은 App_Data 폴더의 UserInfo.mdf 라는 LocalDB 데이터베이스를 지정합니다. 

이번 Tutorial에서 멤버쉽 데이터베이스에 대해 다루지는 않겠지만, 멤버쉽, 인증, 보안에 관한 상세한 내용은 Create an ASP.NET MVC app with auth and SQL DB and deploy to Azure App Service 를 살펴보세요.

이 연결 문자열(name="UserInfoDBContext" 이라 써진 부분) 의 이름은 반드시 DbContext 클래스(상속받은 클래스) 이름과 동일해야 합니다.

using System;
using System.Data.Entity;

namespace GameServer.Models
{
    public class UserInfo
    {
        public int UserID { get; set; }
        public string UserName { get; set; }
        public DateTime LastestConnectDate { get; set; }
    }

    public class UserInfoDBContext : DbContext
    {
        public DbSet<UserInfo> Users { get; set; }
    }
}
사실, UserInfoDBContext 연결 문자열(Connection String)을 추가해 줄 필요는 없었습니다. Entity Framework가 User 디렉토리에 자동으로 DbContext를 상속받은 클래스의 이름을 딴 LocalDB 데이터베이스를 생성 해 줍니다(이 Tutorial에서는 GameServer.Models.UserInfoDBContext)
데이터베이스의 이름은 아무거나 상관 없습니다. .MDF 라는 확장자를 갖기만 하면 됩니다.
