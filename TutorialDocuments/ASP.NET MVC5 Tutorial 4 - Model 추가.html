이번 Tutorial에서는 GameServer 프로젝트의 DataBase를 다루기 위한 클래스를 추가하려고 합니다. 바로 ASP.NET MVC Application의 'Model' 입니다!

Model 클래스를 정의하고 사용하기 위해 .Net Framework 상의 데이터 접근 기술인 Entity Framework 를 사용 하도록 하겠습니다. 
Entity Framework는 (EF라고도 불립니다) Code First 라는 개발 페러다임을 지원 합니다. Code First는 간단한 클래스 포팅으로 'Model' Object들을 생성할 수 있게끔 해줍니다.( 이런 클래스들을 POCO라고 하며 정식 명칭은 "plain-old CLR objects."입니다 )

[ 역주 : '... 뭐 쭉.. 'Model' Class로 DB를 다루기 편하다 라는 뜻입니다.' ]


EF( Entity Framework )에 대해 자세히 알고싶으신 분은 ASP.NET Scaffolding Tutorial을 확인하시기 바랍니다.

Adding Model Classes
Solution Explorer(솔루션 탐색기), 에서 Models 폴더를 우클릭하여 Add -> Class 를 클릭 합니다.

class 의 이름을 'PlayerReplicationInfo' 라고 합시다.
PlayerReplicationInfo class  내용을 다음과 같이 수정 합니다.

아래와 같이 UserInfoDBContext클래스를 추가 해 봅시다.
using System;
using System.Data.Entity;

namespace GameServer.Models
{
    public class UserInfo
    {
        public int UserID { get; set; }
        public string UserName { get; set; }
        public DateTime LastestConnectDate { get; set; }
    }

    public class UserInfoDBContext : DbContext
    {
        public DbSet<UserInfo> Users { get; set; }
    }
}
UserInfoDBContext class 는DB로에 저장된 UserInfo 인스턴스를 저장, 조회, 갱신하는 작업을 처리하는 UserInfo  DB Context 입니다. 
UserInfoDBContext 클래스가 Entity Framework( System.Data.Entity )가 DbContext 클래스의 상속을 받은것에 주목해야 합니다.


DbContext 나 DbSet 클래스를 활용하려면 아래의 using 문구가 들어가야 합니다.

using System.Data.Entity;

수동으로 using 구문을 추가해도 되고, 붉게 밑줄친 코드를 우클릭하여 Resolve -> using System.Data.Entity 을 선택 해도 됩니다.

[역주 : Visual Studio 2015 기준으로 밑줄친 단어에 커서를 옮긴 뒤 Ctrl + . 키를 누르면 해결 가능한 몇가지 예시들을 제공 해 줍니다. 아래 스크린샷을 참조 바랍니다.]



사용하지 않는 using 구문들을 정리할 수 있습니다. 에디트 창에 우클릭하여 Organize Usings, -> Remove Unused Usings 을 눌러보세요.
