이번에는 View Template File을 이용해 HelloWorldController클래스 를 수정하여 캡슐화시킨 HTML Response 처리 방법을 알아보겠습니다.
지저분한 String 처리보다 훨씬 깔끔 해 질겁니다.

Razor view engine.을 사용하여 View Template을 만들겠습니다. Razor 기반의 View Template에는 .cshtml 확장자가 붙습니다.
Razor는 View Template을 만들 때 필요한 숫자 혹은 문자열들을 최소화 시켜 효율적인 Application 구동이 가능하게 합니다.

현재 HelloWorldController::Index() 함수는 Controller 클래스에 하드코딩된 문자열을 리턴하고 있습니다. 
HelloWorldController:: Index() 함수가 View Object를 반환하게 아래와 같이 코드를 수정 합니다.
public ActionResult Index() 
{ 
    return View(); 
}
위의 Index 함수는 Browser의 HTML Response를 생성하는 View Template을 사용 합니다.
Controller Class의 함수들은 일반적으로 위의 예제처럼 ActionResult 를 반환합니다.
string 같은 단순 자료형이 아니라요

Views\HellowWorld 폴더를 우 클릭하여 Add 를 누르고 MVC 5 View Page with (Layout Razor) 를 눌러줍니다.

  

Specify Name for Item 창에 Index라고 적고 OK를 누릅시다!

 

Select a Layout Page 창에서 _Layout.cshtml 를 선택하고 OK를 누릅니다.

 

위의 창 왼쪽에서 Views\Shared 폴더를 선택 했습니다. 만약 당신의 Custom Lyaout 파일이 있다면 그걸 선택해도 됩니다.
이후의 Tutorial에서 Layout 파일에 대해 다룰 에정입니다.

MvcMovie\Views\HelloWorld\Index.cshtml 파일이 만들어졌습니다. ( 제가 진행 할 예제에서는 GameServer\Views\HelloWorld\index.cshtm ) 입니다.

화제를 바꿔, Application을 Ctrl + F5로 실행하고, HelloWorld Controller를 호출 해 보세요. (http://localhost:xxxx/HelloWorld).
Controller의 Index 함수는 현재까지는 아무 역할을 하지 못합니다.단순히 return View() 구문만 수행하고 있지만, 이는 Rendering을 위해서 View Template을 사용해야 한다는 위미를 내포합니다.
명시적인 View Template 파일을 지정이 없다면 ASP.NET MVC 는 Controllers\HelloWorldController.cs 에 대응하는 Views\HelloWorld 폴더의 Index.cshtml 파일을 디폴트로 사용합니다.

View & Layout 변경

검은색 상단바에 적힌 "Application name" 에 대한 링크를 변경하겠습니다. 모든 Page에서 공용으로 사용되며 모든 페이지에서 단 한 곳에만 존재합니다. C++의 extern과 같은 개념으로 받아 들이시면 됩니다.
Solution Explorer  의 /Views/Shared 폴더에서  _Layout.cshtml 파일을 엽니다. Layout Page라고 하며,  모든 page에서 사용합니다.

Visual Sutdio 2015 버전의 ASP.NET MVC5 Web Application에서는 _Layout.cshtml 파일의 내용이 약간 다릅니다. 참고 부탁드립니다.


Layout template 은 특정 HTML Container layout을 한 곳에 박고, 당신의 WebSite 여러 군데에서 공유 시킬 수 있습니다.
@RenderBody() 라고 적힌 구문을 찾아 보세요. RenderBody 는 ViewPage를 PlaceHolder 위치( Layout의 위치 종류 ) 에 렌더링 해 주는 역할을 합니다. 
DirectX로 따지면 Default RenderTarget에 렌더링을 하는 DrawPrimitive 역할을 한다고 볼 수 있습니다.


예를 들어  About 링크를 누르게 되면, Views\Home\About.cshtml View Template이 RenderBody 함수에서 렌더링 됩니다.

Controller 에서 View로 Data 전송


DB와 Model에 대해서 얘기하기 전에, Controller에서 View로 어떻게 정보를 보내는지 알아보겠습니다. Contrller 클래스는 브라우저에 입력한 URL의 요청에 대한 응답에 의해 실행 됩니다. URL 요청에 대해 DB로 부터 데이터를 조회하고 어떠한 종류의 Response를 브라우저에 다시 보내줄 지 결정해 주는 것이 Controller 클래스 입니다. 
이 때 Controller의 응답은 View Template입니다. 

이 순서를 간단하게 요약하자면 아래와 같습니다.

     1. [Client] URL 요청 to Server
     2. [Server] Controller 실행
     3. [Server] Controller 분석 및 적합한 View Template 반환
     4. [Server] URL Response HTML Type to Client 
     5. [Client] Browser가 HTML Response를 분석하여 화면에 표시

Controller는 View Template이 브라우저에 Response 생성에 사용 할 데이터나 객체를 제공합니다.  View Template에 대한 최고의 사용방법은 다음과 같습니다.

View에서는 Logic을 수행하지 않고 DB와도 직접적인 연계를 하지 않는다.

이는 View Template을 "Read Only"로서 활용하는 것과 같다고 봐도 무방합니다.
View Template은 Controller가 제공해준 Data만 사용해야 합니다.  "데이터와 로직, 렌더링의 분리"는 MVC의 기본 철학이기도 하며
깨끗한 코딩을 가능하게 합니다. 즉, 유지보수에 뛰어난 장점이 있습니다.

현재  HelloWorldController  클래스의 Welcome 함수는 name과 id를 변수로 받으며 이를 Browerser에 바로 리턴 해 주고 있습니다.
Controller가 문자열이 아닌 View Template을 리턴하도록 해 보겠습니다. View Template은 Dynamic Response를 생성해 주는데, 이를 위해서는 View Template에 적당한 값을 전달 해 주어야 합니다. View Template이 접근 가능한 ViewBag 이라는 Object를 이용 해 보겠습니다.

HelloWorldController.cs 로 돌아갑니다.  Welcome 함수 내부에 ViewBag object 에 Message 와 id 값을 추가 합니다. ViewBag 은 동적 객체입니다. 원하는 모든 값을 집어 넣을 수 있다는 뜻입니다( 특별한 자료형이 존재하지 않습니다. JSON 혹은 JavaScript의 자료형을 생각하시면 될 것 같습니다. ).  ASP.NET MVC model binding system은 쿼리에 주어진 매개변수 이름을 이용해서 자동으로 메서드의 매개변수들과  매핑 해 줍니다. 수정 된 HelloWorldController.cs 의 모습은 다음과 같습니다.
using System.Web;using System.Web.Mvc;

namespace MvcMovie.Controllers{
    public class HelloWorldController : Controller
    {
        public ActionResult Index()
        {
            return View();
        }

        public ActionResult Welcome(string name, int numTimes = 1)
        {
            ViewBag.Message = "Hello " + name;
            ViewBag.NumTimes = numTimes;

            return View();
        }
    }}
이제 ViewBag 오브젝트는 View Template에 전달할 데이터를 포함합니다. 다음으로 Welcome View Template이 필요 합니다!

Welcome.cshtml 파일을 다음 코드로 수정 해 주세요. NumTimes 만큼 Message를 출력 하는 내용입니다.

@{
    ViewBag.Title = "Welcome";
}

<h2>Welcome</h2>

<ul>

    {
        <li>@ViewBag.Message</li>
    }
</ul>
Ctrl + F5를 눌러 Browser 실행 후 다음과 같은 URL로 접속 해 보세요 ( 
http://localhost:xx/HelloWorld/Welcome?name=Scott&numtimes=4

위 URL을 Browser에 입력한 이후의 프로세스는 다음과 같습니다.

1. Data( Name = Scott, NumTimes = 4 ) 의 값이 model binder를 통해 HelloWorldController의 Welcome 함수로 전달.
2. HelloWorldController.Welcome() 함수는 그 데이터를 ViewBag object에 저장후 View로 넘김
3. View가 Browser가 인식 가능한 HTML형태로 렌더링 


위의 예제에서 ViewBag object를 통해 Controller -> View 데이터 전달하는 방법을 살펴 봤습니다.
