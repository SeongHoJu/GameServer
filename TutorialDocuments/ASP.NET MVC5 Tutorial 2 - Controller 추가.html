MVC는 Model + View + Controller의 약자로, 좋은 설계, 효율적인 테스트 및 유지보수가 용이한 Developing 패턴입니다.
  • Models: 데이터를 상징하는 Class입니다.  [ In Game = Player Stat ]
  • Views: HTML Request를 생성하는 Template입니다. [ In Game = Pawn ]
  • Controllers: HTML Request에 대응해서 Model 데이터를 검색해 응답하는 View Template을 지정합니다. [ In Game = Controller ]
ASP.NET MVC5 Tutorial 시리즈에서 MVC 개념을 사용하여 Tutorial을 진행 할 예정입니다.


Solution Explorer 탭에서 Controller를 추가 합니다. 

Controller의 이름은 HelloWorldController로 합니다.

Solution Explorer에서 Controllers\HelloWorldController.cs 와 Views\HelloWorld가 생성된 것을 확인할 수 있습니다.

HelloWorldController.cs의 내용을 다음과 같이 수정합니다.
using System.Web;using System.Web.Mvc; 
 
namespace MvcMovie.Controllers 
{ 
    public class HelloWorldController : Controller 
    { 
        // 
        // GET: /HelloWorld/ 
 
        public string Index() 
        { 
            return "This is my <b>default</b> action..."; 
        } 
 
        // 
        // GET: /HelloWorld/Welcome/ 
 
        public string Welcome() 
        { 
            return "This is the Welcome action method..."; 
        } 
    } 
}
Controller에 등록된 함수들은 http주소로 접근 가능합니다. 함수 이름이 곧 주소가 된다고 볼 수 있습니다.
localhost:xxxx/HelloWorld 입력은 default값으로 index()에 해당하는 함수 반환값을 보여줍니다. 

Ctrl + F5를 눌러서 Application을 실행하고 기본값인 localhost:xxxx/에 추가 HelloWorld를 입력하면 HelloWorldController.cs의 Index()함수가 반환하는 string값을 확인할 수 있습니다.


[Controller]/[ActionName]/[Parameters]
URL 주소는 위와 같은 형식으로 구분 지을 수 있습니다. 

이는 다음의 코드와 같습니다. 
[Class]/[Function]/[Arguments]
[클래스 이름]/[함수 이름]/[매개변수]
[Player]/[Attack]/[TargetPlayer,Damage]


public class Player
{
     public void Attack( Player TargetPlayer, int Damage )
     {
          TargetPlayer.HP -= Damage;
     } 
}

App_Start/RouteConfig.cs 파일을 보면 다음과 같은 함수가 있습니다.
public static void RegisterRoutes(RouteCollection routes){
    routes.IgnoreRoute("{resource}.axd/{*pathInfo}");

    routes.MapRoute(
        name: "Default",
        url: "{controller}/{action}/{id}",
        defaults: new { controller = "Home", action = "Index", id = UrlParameter.Optional }
    );}
Ctrl + F5로 프로그램을 시작하고 아무 URL도 넣지 않으면 기본적으로 [ HomeController.cs ] [ Index() ][ Optional ] 이런식의 호출이 이루어지게 됩니다.
위에서 말한대로 [클래스 이름][함수 이름][매개변수] 와같은 형태의 호출입니다.
매개변수를 넣는 예제 입니다. (/HelloWorld/Welcome?name=Scott&numtimes=4). 

HelloWorldController.cs의 Welcome 함수를 아래와 같이 수정합니다. 
public string Welcome(string name, int numTimes = 1) {
     return HttpUtility.HtmlEncode("Hello " + name + ", NumTimes is: " + numTimes);}

Security Note: 위의 코드는 HttpServerUtility.HtmlEncode Method를 사용 합니다. Java Script의 공격으로 부터 Application을 방어 합니다. 자세한 사용법은  How to: Protect Against Script Exploits in a Web Application by Applying HTML Encoding to Strings. 참조 부탁 드립니다.
Ctrl + F5를 눌러 Application을 실행하고 다음과 같이 URL을 입력 합니다. (http://localhost:xxxx/HelloWorld/Welcome?name=Scott&numtimes=4). 
URL 입력시 name 과 numtimes값을 바꾸어서 실행할 수 있습니다.ASP.NET MVC model binding system 은 자동으로 URL의 쿼리문을 매개 변수로 매핑하여 처리 합니다.
위의 URL에서 '?'는 쿼리 문자열을 구분하는 문자입니다.

Welcome 함수를 다음과같이 수정 합니다.
public string Welcome(string name, int ID = 1){
    return HttpUtility.HtmlEncode("Hello " + name + ", ID: " + ID);}
Ctrl + F5를 눌러 프로그램을 실행하고 다음 URL을 입력 해 보세요:  http://localhost:xxx/HelloWorld/Welcome/3?name=Rick


세번째 URL Parameter[ /3?name=Rick  ] 에 해당하는 string중 '/3' 값은 routes.mapRoute로 지정한 url: "{controller}/{action}/{id}" 중 {id} 값에 대응 됩니다.

public static void RegisterRoutes(RouteCollection routes){
    routes.IgnoreRoute("{resource}.axd/{*pathInfo}");

    routes.MapRoute(
        name: "Default",
        url: "{controller}/{action}/{id}",
        defaults: new { controller = "Home", action = "Index", id = UrlParameter.Optional }
    );}
ASP.NET MVC Application에서는 쿼리 문자열로 URL을 전달하는 방식보다 [/3?name=Rick ]
MapRoute에 등록한 양식을 사용하여 URL을 전달하는 방식이 더 일반적입니다. [아래의 예제 참조]


App_Start/RouteConfig.cs 파일을 다음과 같이 수정 하세요.
public class RouteConfig{
   public static void RegisterRoutes(RouteCollection routes)
   {
      routes.IgnoreRoute("{resource}.axd/{*pathInfo}");

      routes.MapRoute(
          name: "Default",
          url: "{controller}/{action}/{id}",
          defaults: new { controller = "Home", action = "Index", id = UrlParameter.Optional }
      );

      routes.MapRoute(
           name: "Hello",
           url: "{controller}/{action}/{name}/{id}"
       );
   }}
이제 Ctrl + F5를 눌러 Application을 실행하고 URL을 다음과 같이 입력 합니다.
/localhost:XXX/HelloWorld/Welcome/Scott/3.

MapRoute로 추가한 URL양식이 잘 작동하는것을 확인할 수 있습니다.


      routes.MapRoute(
           name: "Hello",
           url: "{controller}/{action}/{name}/{id}"
      );
위의 함수에서 name은 URL을 입력과는 무관한 값입니다. Application [Code 영역] 에서 Route를 찾을 필요가 있을때 사용됩니다.
동일한 이름으로 MapRoute를 추가하는것은 불가능하며 guid와 같은 형식으로 받아들이시면 됩니다.

많은 MVC Application에서 "default" Route는 문제없이 작동합니다. 
이후 Model Binder를 사용하여 데이터를 전달하는 방법이 Tutorial의 후반부에서 소개 될 예정이며, "default" Route를 수정할 필요는 없습니다.

Tutorial2. Controller 추가 에서는 Controller가 View 영역까지 처리하며, HTML을 직접 반환하고 있었습니다.
이는 직관적인 Controller의 역할을 보여주기 위한 단순 예제에 불과합니다.

일반적인 MVC Application은 HTML Request( [Controller]/[ActionName]/[Parameters]  형식의 URL) 에 대해 View Template file을 Response 처리 합니다.
